How to get game files?

1. Download game files from the official source

Guide for Steam
Zomdroid requires Project Zomboid files for Linux in order to run. If you're using Linux, simply download the game as usual through the Steam client. If you're using Windows or MacOS, follow these steps:
  1. Open the Steam console. One way to do this is by typing steam://open/console into your browser’s address bar
  2. Enter the following command: download_depot 108600 108603. This will download the latest stable version of Project Zomboid for Linux. If you want to download a specific version, you'll also need to provide a manifest id. For example, to download the unstable version from March 24, 2025, use: download_depot 108600 108603 2006259071304963476. You can find a list of all manifests on the SteamDB website (not affiliated with Steam or Valve). Steam only allows downloading manifests that belong to the currently active branch. For example, if you want to download files from the Build 42 branch, you must first opt into the Unstable branch in your Steam game's properties.

2. Add files to a .zip archive

Zomdroid requires the .zip archive to contain the game files at the top level — not inside a folder.

Guide for Steam
Navigate to folder containing game files (folder with ProjectZomboid64 executable). On Linux it will most likely be located at ~/.local/share/Steam/steamapps/common/ProjectZomboid/projectzomboid. On Windows or macOS you need to note the depot download directory from the previous step (it will appear in the console once the download is finished). E.g. on Windows it is usually C:\Program Files (x86)\Steam\steamapps\content\app_108600\depot_108603\projectzomboid. Select all files (Ctrl + A or Command + A on MacOS) and add them to a .zip archive

3. Copy the .zip archive to your mobile device

Copy the .zip archive from the previous step to any folder on your mobile device. You can later use that file to create a new game instance. Once the game instance is created, you can delete the .zip archive to free up space.

Native libraries (Build 41 multiplayer)

To use multiplayer in Build 41, additional native libraries are required. These libraries are officially distributed by the Project Zomboid developers as part of Build 42.12 (Linux version).

Only two native libraries are required:

  • libRakNet64.so
  • libZNetNoSteam64.so

Download these files from the Linux Build 42.12 distribution, place both files into a single .zip archive, and upload this archive when creating a new game instance.

The .zip archive should contain the two .so files at the top level (not inside a folder).

You can upload the native libraries archive together with the main game files in the corresponding fields when creating a new game instance.

Renderers and drivers

Zomdroid uses renderers to translate the desktop graphics of Project Zomboid into graphics compatible with Android devices.

The launcher provides two rendering solutions:

  • GL4ES
  • ZINK (ZINK_ZFA / ZINK_OSMESA)

GL4ES

GL4ES is the most stable renderer for Build 41. However, in Build 42 it often produces graphical artifacts, as it cannot fully translate the updated rendering pipeline of the game.

ZINK

ZINK provides the best results with Build 42 and usually eliminates the graphical artifacts seen with GL4ES.

ZINK offers two Vulkan drivers:

  • System Vulkan driver
  • Freedreno (Turnip / Mesa)

ZINK primarily works on Snapdragon devices (Adreno GPUs). On MediaTek (Helio, Dimensity) and other platforms (Exynos, Kirin, etc.), ZINK often does not work or may crash.

The ideal configuration for both Build 41 and Build 42 is ZINK + Freedreno. However, Freedreno is supported only on certain Snapdragon devices with Adreno GPUs up to the 7xx series. Newer Snapdragon platforms (Elite / Gen 5) are not yet supported.
