How to get game files?

1. Download game files from the official source

Guide for DepotDownloader

Zomdroid requires Project Zomboid files for Linux to run. The easiest way to get them on Windows is via DepotDownloader — a tool that lets you download any specific build directly from Steam.

You need to own Project Zomboid on Steam before downloading.

  1. Download DepotDownloader. For Windows, grab DepotDownloader-windows-x64.zip.
  2. Extract it anywhere, for example: C:\DepotDownloader
  3. Open the folder, right-click → Open in Terminal / CMD
  4. Run the following command as a base, replacing YOURUSERNAME and YOURPASSWORD with your Steam credentials:

    ./DepotDownloader -app 108600 -depot 108603 -manifest MANIFESTNUMBER -username YOURUSERNAME -password YOURPASSWORD

    The exact command including the manifest number and optional -beta flag (e.g. -beta unstable or -beta legacy_42_12) can be copied directly from SteamDB — select DepotDownloader from the "Copy format" menu and click the copy icon next to the desired build. Steam Mobile authentication may be required.

2. Add files to a .zip archive

Zomdroid requires the .zip archive to contain the game files at the top level — not inside a folder.

Guide for Steam

Navigate to folder containing game files (folder with ProjectZomboid64 executable). On Linux it will most likely be located at ~/.local/share/Steam/steamapps/common/ProjectZomboid/projectzomboid. On Windows or macOS you need to note the depot download directory from the previous step (it will appear in the console once the download is finished). E.g. on Windows it is usually C:\Program Files (x86)\Steam\steamapps\content\app_108600\depot_108603\projectzomboid. Select all files (Ctrl + A or Command + A on MacOS) and add them to a .zip archive

3. Copy the .zip archive to your mobile device

Copy the .zip archive from the previous step to any folder on your mobile device. You can later use that file to create a new game instance. Once the game instance is created, you can delete the .zip archive to free up space.

Native libraries (Build 41 multiplayer)

To use multiplayer in Build 41, additional native libraries are required. These libraries are officially distributed by the Project Zomboid developers as part of Build 42.12 (Linux version).

⚠️Important: the required files are located only inside the android folder. Only files from this folder should be used — other files are not considered native libraries.

Only two native libraries are required:

  • libRakNet64.so
  • libZNetNoSteam64.so

Download these files from the Linux Build 42.12 distribution, place both files into a single .zip archive, and upload this archive when creating a new game instance.

The .zip archive should contain the two .so files at the top level (not inside a folder).

You can upload the native libraries archive together with the main game files in the corresponding fields when creating a new game instance.

Renderers and drivers

Zomdroid uses renderers to translate the desktop graphics of Project Zomboid into graphics compatible with Android devices.

The launcher provides two rendering solutions:

  • GL4ES
  • ZINK (ZINK_ZFA / ZINK_OSMESA)

GL4ES

GL4ES is the most stable renderer for Build 41. However, in Build 42 it often produces graphical artifacts, as it cannot fully translate the updated rendering pipeline of the game.

ZINK

ZINK provides the best results with Build 42 and usually eliminates the graphical artifacts seen with GL4ES.

ZINK offers a few Vulkan drivers:

  • System Vulkan driver
  • Freedreno
  • Turnip

All Freedreno and Turnip drivers listed above are builds of the same open-source Mesa Vulkan driver for Adreno GPUs. Turnip is the official name of the Vulkan component within the Mesa Freedreno driver stack — the names are sometimes used interchangeably. Different builds are optimized for different Snapdragon generations, which is why the launcher offers several driver options.

ZINK primarily works on Snapdragon devices (Adreno GPUs). On MediaTek Dimensity, ZINK may work via the ANGLE graphics driver, however enabling ANGLE on Dimensity devices is often non-trivial — it rarely works through developer settings alone and may require third-party tools. On MediaTek Helio, ZINK is not supported, but some Helio devices may be able to run earlier, less demanding Build 42 versions (such as 42.6) via GL4ES — no reliable solution exists yet, but this may change in the future. For other processors (Exynos, Kirin, etc.), there is limited information available — ZINK often does not work or may crash.

The recommended configuration for Build 41 is GL4ES. For Build 42, the recommended configuration is ZINK + a Freedreno driver. Freedreno drivers are supported only on Snapdragon devices with Adreno GPUs. Different driver versions target different hardware generations:

  • Freedreno — stable, recommended for Adreno 6xx and 7xx (Snapdragon up to 7 Gen 1)
  • Freedreno 8xx — experimental, for Adreno 8xx (Snapdragon 8 Gen 3 / Gen 4 / Gen 5 / Elite)
  • Freedreno 830/840 — experimental, specifically for Adreno 830/840 (Snapdragon 8 Elite / Elite 2, e.g. Galaxy S25/S26 series)

Known issues

Build 42.12+ — crashes during world generation

Starting from approximately Build 42.12, the game may crash during world creation — specifically when loading into the game after character creation. This appears to be a game-side issue unrelated to the device or driver, as it has been observed on multiple different devices and configurations.

This is not a permanent failure. If the game crashes at this stage, simply restart Zomdroid and try again. Most users need 2–4 attempts before the world loads successfully for the first time. Once the world has been created, subsequent loads of the same save are usually stable.

Graphical mods compatibility

Some graphical mods — such as retextures of clothing, hair, or vehicles — may not display correctly or may not work at all. This is a known limitation of the graphics translation layer used to run the game on Android, and is not specific to any particular device or driver.

GUI disappears near dismantled vehicles

The game UI may disappear if there is a vehicle nearby with 3 or more wheels removed. The cause is a rendering issue triggered by the vehicle's state. Workaround: avoid removing more than 2 wheels from any single vehicle. If the UI has already disappeared, moving away from the vehicle or reloading the save should restore it.
