如何获取游戏文件？

1. 从官方渠道下载游戏文件

DepotDownloader 使用指南

Zomdroid 需要 Linux 版本的《僵尸毁灭工程》文件才能运行。在 Windows 上获取这些文件最简单的方式是使用 DepotDownloader —— 一个可以直接从 Steam 下载指定版本的工具。

下载前您需要在 Steam 上拥有《僵尸毁灭工程》。

  1. 下载 DepotDownloader。 Windows 用户请选择 DepotDownloader-windows-x64.zip。
  2. 解压到任意位置，例如：C:\DepotDownloader
  3. 打开文件夹，右键单击 → 在终端/CMD中打开
  4. 使用以下命令作为基础，将 YOURUSERNAME 和 YOURPASSWORD 替换为您的 Steam 账号信息：

    ./DepotDownloader -app 108600 -depot 108603 -manifest MANIFESTNUMBER -username YOURUSERNAME -password YOURPASSWORD

    包含 manifest 编号和可选 -beta 标志 （例如 -beta unstable 或 -beta legacy_42_12）的完整命令 可直接从 SteamDB 复制 —— 在「Copy format」下拉菜单中选择 DepotDownloader，然后点击所需版本旁边的复制图标。 可能需要通过 Steam 手机端进行验证。

2. 将文件添加到 .zip 压缩包

Zomdroid 要求 .zip 压缩包的根目录直接包含游戏文件 —— 不能放在子文件夹内。

Steam 使用指南

进入包含游戏文件的文件夹（含 ProjectZomboid64 可执行文件的文件夹）。 Linux 系统通常位于 ~/.local/share/Steam/steamapps/common/ProjectZomboid/projectzomboid。 Windows 或 macOS 用户请记录上一步的 depot 下载目录 （下载完成后会显示在控制台中）。Windows 上通常为 C:\Program Files (x86)\Steam\steamapps\content\app_108600\depot_108603\projectzomboid。 全选所有文件（Ctrl + A 或 macOS 上的 Command + A），然后添加到 .zip 压缩包中。

3. 将 .zip 压缩包复制到手机

将上一步创建的 .zip 压缩包复制到手机上的任意文件夹。之后可以用该文件创建新的游戏实例。游戏实例创建完成后，可以删除 .zip 压缩包以释放空间。

原生库（Build 41 联机）

在 Build 41 中使用联机功能需要额外的原生库。 这些库由《僵尸毁灭工程》开发者作为 Build 42.12（Linux 版本） 的一部分官方发布。

只需要两个原生库：

  • libRakNet64.so
  • libZNetNoSteam64.so

从 Linux Build 42.12 发行版中下载这两个文件， 将它们放入一个 .zip 压缩包， 并在创建新游戏实例时上传该压缩包。

.zip 压缩包应将两个 .so 文件 直接放在根目录（不在文件夹内）。

在创建新游戏实例时，可以将原生库压缩包与主游戏文件一起 上传到对应字段中。

渲染器与驱动

Zomdroid 使用渲染器将《僵尸毁灭工程》的桌面端图形 转换为与 Android 设备兼容的图形。

启动器提供两种渲染方案：

  • GL4ES
  • ZINK（ZINK_ZFA / ZINK_OSMESA）

GL4ES

GL4ES 是 Build 41 最稳定的渲染器。 但在 Build 42 中，它常常产生图形瑕疵， 因为它无法完全转换游戏更新后的渲染管线。

ZINK

ZINK 在 Build 42 上效果最佳， 通常能消除 GL4ES 产生的图形瑕疵。

ZINK 提供以下几种 Vulkan 驱动：

  • 系统 Vulkan 驱动
  • Freedreno
  • Turnip

上述所有 Freedreno 和 Turnip 驱动都是同一款开源 Mesa Vulkan 驱动（适用于 Adreno GPU）的不同版本。Turnip 是 Mesa Freedreno 驱动栈中 Vulkan 组件的正式名称 —— 这两个名称有时可互换使用。 不同版本针对不同的骁龙处理器代进行了优化， 这也是为什么启动器提供多个驱动选项。

ZINK 主要适用于骁龙设备（Adreno GPU）。在联发科 Dimensity 上， ZINK 可通过 ANGLE 图形驱动工作，但在 Dimensity 设备上启用 ANGLE 往往并不简单 —— 仅靠开发者选项很少能直接成功，可能需要第三方工具。联发科 Helio 不支持 ZINK， 但部分 Helio 设备可能能够通过 GL4ES 运行较早、要求较低的 Build 42 版本（如 42.6）—— 目前尚无可靠方案，但未来情况可能改变。 对于其他处理器（Exynos、麒麟等），相关信息较少 —— ZINK 通常无法正常工作或可能崩溃。

Build 41 的推荐配置为 GL4ES。 Build 42 的推荐配置为 ZINK + Freedreno 驱动。 Freedreno 驱动仅支持搭载 Adreno GPU 的骁龙设备。 不同驱动版本适用于不同的硬件代：

  • Freedreno —— 稳定版，推荐用于 Adreno 6xx 和 7xx（骁龙 7 Gen 1 及以下）
  • Freedreno 8xx —— 实验性，适用于 Adreno 8xx（骁龙 8 Gen 3 / Gen 4 / Gen 5 / Elite）
  • Freedreno 830/840 —— 实验性，专为 Adreno 830/840（骁龙 8 Elite / Elite 2，如 Galaxy S25/S26 系列）

已知问题

Build 42.12+ —— 创建世界时崩溃

从大约 Build 42.12 开始，游戏可能在创建世界时崩溃 —— 具体发生在创建角色后加载游戏的阶段。 这似乎是游戏本身的问题，与设备或驱动无关， 因为在多种不同设备和配置上均有发现。

这不是永久性故障。 如果游戏在此阶段崩溃， 只需重启 Zomdroid 再试一次。大多数用户需要 2–4 次尝试 才能成功首次加载世界。世界创建完成后， 后续加载同一存档通常会很稳定。

图形 MOD 兼容性

部分图形 MOD —— 例如服装、发型或载具的重贴图 —— 可能无法正确显示或完全无法使用。 这是在 Android 上运行游戏所用图形转换层的已知限制， 与具体设备或驱动无关。

在拆解车辆附近 GUI 消失

如果附近有被拆除了 3 个或更多车轮 的载具，游戏界面可能会消失。 原因是载具状态触发的渲染问题。 解决方法：避免从同一辆载具上拆除超过 2 个车轮。 如果界面已经消失，远离该载具或重新加载存档应该能恢复。
