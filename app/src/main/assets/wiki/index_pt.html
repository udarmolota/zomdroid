Como obter os arquivos do jogo?

1. Baixar os arquivos do jogo da fonte oficial

Guia para o DepotDownloader

O Zomdroid requer os arquivos do Project Zomboid para Linux. A maneira mais fácil de obtê-los no Windows é usando o DepotDownloader — uma ferramenta que permite baixar qualquer versão específica diretamente do Steam.

Você precisa possuir o Project Zomboid no Steam antes de fazer o download.

  1. Baixe o DepotDownloader. Para Windows, pegue o DepotDownloader-windows-x64.zip.
  2. Extraia em qualquer lugar, por exemplo: C:\DepotDownloader
  3. Abra a pasta, clique com o botão direito → Abrir no Terminal / CMD
  4. Use o seguinte comando como base, substituindo SEUUSUARIO e SUASENHA pelas suas credenciais do Steam:

    ./DepotDownloader -app 108600 -depot 108603 -manifest NUMERODOMANIFEST -username SEUUSUARIO -password SUASENHA

    O comando exato com o número do manifest e o flag opcional -beta (ex.: -beta unstable ou -beta legacy_42_12) pode ser copiado diretamente do SteamDB — selecione DepotDownloader no menu "Copy format" e clique no ícone de copiar ao lado da versão desejada. A autenticação pelo Steam Mobile pode ser necessária.

2. Adicionar os arquivos a um arquivo .zip

O Zomdroid requer que o arquivo .zip contenha os arquivos do jogo no nível raiz — não dentro de uma pasta.

Guia para o Steam

Navegue até a pasta contendo os arquivos do jogo (pasta com o executável ProjectZomboid64). No Linux, ela geralmente está em ~/.local/share/Steam/steamapps/common/ProjectZomboid/projectzomboid. No Windows ou macOS, anote o diretório de download do depot da etapa anterior (ele aparecerá no console ao término do download). No Windows, geralmente é C:\Program Files (x86)\Steam\steamapps\content\app_108600\depot_108603\projectzomboid. Selecione todos os arquivos (Ctrl + A ou Command + A no macOS) e adicione-os a um arquivo .zip.

3. Copiar o arquivo .zip para o dispositivo móvel

Copie o arquivo .zip da etapa anterior para qualquer pasta no seu dispositivo móvel. Você poderá usar esse arquivo para criar uma nova instância do jogo. Após criar a instância, você pode excluir o arquivo .zip para liberar espaço.

Bibliotecas nativas (multiplayer do Build 41)

Para usar o multiplayer no Build 41, são necessárias bibliotecas nativas adicionais. Essas bibliotecas são distribuídas oficialmente pelos desenvolvedores do Project Zomboid como parte do Build 42.12 (versão Linux).

⚠️Importante: os arquivos necessários estão localizados apenas na pasta android. Apenas arquivos desta pasta devem ser usados — arquivos de outras pastas não são considerados bibliotecas nativas.

Apenas duas bibliotecas nativas são necessárias:

  • libRakNet64.so
  • libZNetNoSteam64.so

Baixe esses arquivos da distribuição Linux do Build 42.12, coloque ambos em um único arquivo .zip e faça o upload desse arquivo ao criar uma nova instância do jogo.

O arquivo .zip deve conter os dois arquivos .so no nível raiz (não dentro de uma pasta).

Você pode fazer o upload do arquivo de bibliotecas nativas junto com os arquivos principais do jogo nos campos correspondentes ao criar uma nova instância do jogo.

Renderizadores e drivers

O Zomdroid usa renderizadores para traduzir os gráficos desktop do Project Zomboid em gráficos compatíveis com dispositivos Android.

O launcher oferece duas soluções de renderização:

  • GL4ES
  • ZINK (ZINK_ZFA / ZINK_OSMESA)

GL4ES

O GL4ES é o renderizador mais estável para o Build 41. No entanto, no Build 42 frequentemente produz artefatos gráficos, pois não consegue traduzir completamente o pipeline de renderização atualizado do jogo.

ZINK

O ZINK oferece os melhores resultados com o Build 42 e geralmente elimina os artefatos gráficos vistos com o GL4ES.

O ZINK oferece alguns drivers Vulkan:

  • Driver Vulkan do sistema
  • Freedreno
  • Turnip

Todos os drivers Freedreno e Turnip listados acima são versões do mesmo driver Mesa Vulkan de código aberto para GPUs Adreno. Turnip é o nome oficial do componente Vulkan dentro da pilha de drivers Mesa Freedreno — os nomes às vezes são usados de forma intercambiável. Diferentes versões são otimizadas para diferentes gerações de Snapdragon, por isso o launcher oferece várias opções de drivers.

O ZINK funciona principalmente em dispositivos Snapdragon (GPUs Adreno). No MediaTek Dimensity, o ZINK pode funcionar via driver gráfico ANGLE, porém habilitar o ANGLE em dispositivos Dimensity frequentemente não é trivial — raramente funciona apenas pelas configurações do desenvolvedor e pode exigir ferramentas de terceiros. No MediaTek Helio, o ZINK não é suportado, mas alguns dispositivos Helio podem rodar versões anteriores e menos exigentes do Build 42 (como a 42.6) via GL4ES — ainda não existe uma solução confiável, mas isso pode mudar no futuro. Para outros processadores (Exynos, Kirin, etc.), há informações limitadas disponíveis — o ZINK frequentemente não funciona ou pode travar.

A configuração recomendada para o Build 41 é GL4ES. Para o Build 42, a configuração recomendada é ZINK + um driver Freedreno. Os drivers Freedreno são suportados apenas em dispositivos Snapdragon com GPUs Adreno. Diferentes versões de drivers são destinadas a diferentes gerações de hardware:

  • Freedreno — estável, recomendado para Adreno 6xx e 7xx (Snapdragon até 7 Gen 1)
  • Freedreno 8xx — experimental, para Adreno 8xx (Snapdragon 8 Gen 3 / Gen 4 / Gen 5 / Elite)
  • Freedreno 830/840 — experimental, especificamente para Adreno 830/840 (Snapdragon 8 Elite / Elite 2, ex.: Galaxy S25/S26)

Problemas conhecidos

Build 42.12+ — travamentos durante a geração do mundo

A partir do Build 42.12, o jogo pode travar durante a criação do mundo — especificamente ao carregar o jogo após a criação do personagem. Isso parece ser um problema do próprio jogo, não relacionado ao dispositivo ou driver, pois foi observado em vários dispositivos e configurações diferentes.

Isso não é uma falha permanente. Se o jogo travar nessa etapa, simplesmente reinicie o Zomdroid e tente novamente. A maioria dos usuários precisa de 2 a 4 tentativas antes de carregar o mundo com sucesso pela primeira vez. Após a criação do mundo, os carregamentos subsequentes do mesmo save geralmente são estáveis.

Compatibilidade de mods gráficos

Alguns mods gráficos — como retexturas de roupas, cabelos ou veículos — podem não ser exibidos corretamente ou podem não funcionar de jeito nenhum. Esta é uma limitação conhecida da camada de tradução gráfica usada para rodar o jogo no Android, e não é específica de nenhum dispositivo ou driver em particular.

Interface desaparece perto de veículos desmontados

A interface do jogo pode desaparecer se houver um veículo próximo com 3 ou mais rodas removidas. A causa é um problema de renderização ativado pelo estado do veículo. Solução alternativa: evite remover mais de 2 rodas de um único veículo. Se a interface já desapareceu, afastar-se do veículo ou recarregar o save deve restaurá-la.
