Как получить файлы игры?

1. Скачать файлы игры из официального источника

Руководство по DepotDownloader

Zomdroid требует файлы Project Zomboid для Linux. Самый простой способ получить их на Windows — использовать DepotDownloader — инструмент, позволяющий скачать любую конкретную сборку напрямую из Steam.

Перед загрузкой вы должны владеть Project Zomboid в Steam.

  1. Скачайте DepotDownloader. Для Windows выберите DepotDownloader-windows-x64.zip.
  2. Распакуйте куда угодно, например: C:\DepotDownloader
  3. Откройте папку, правая кнопка мыши → Открыть в терминале / CMD
  4. Используйте следующую команду как основу, заменив YOURUSERNAME и YOURPASSWORD на ваши данные Steam:

    ./DepotDownloader -app 108600 -depot 108603 -manifest MANIFESTNUMBER -username YOURUSERNAME -password YOURPASSWORD

    Точную команду с номером манифеста и опциональным флагом -beta (например -beta unstable или -beta legacy_42_12) можно скопировать прямо с SteamDB — выберите DepotDownloader в меню «Copy format» и нажмите значок копирования рядом с нужной сборкой. Может потребоваться подтверждение через мобильное приложение Steam.

2. Добавить файлы в .zip архив

Zomdroid требует, чтобы файлы игры находились в корне .zip архива — не внутри папки.

Руководство для Steam

Перейдите в папку с файлами игры (папка с исполняемым файлом ProjectZomboid64). На Linux она обычно находится по адресу ~/.local/share/Steam/steamapps/common/ProjectZomboid/projectzomboid. На Windows или macOS используйте директорию загрузки depot из предыдущего шага (она появится в консоли после завершения загрузки). На Windows это обычно C:\Program Files (x86)\Steam\steamapps\content\app_108600\depot_108603\projectzomboid. Выделите все файлы (Ctrl + A или Command + A на macOS) и добавьте их в .zip архив.

3. Скопировать .zip архив на мобильное устройство

Скопируйте .zip архив из предыдущего шага в любую папку на вашем мобильном устройстве. Позже вы сможете использовать этот файл для создания нового экземпляра игры. После создания экземпляра .zip архив можно удалить, чтобы освободить место.

Нативные библиотеки (мультиплеер Build 41)

Для использования мультиплеера в Build 41 требуются дополнительные нативные библиотеки. Эти библиотеки официально распространяются разработчиками Project Zomboid в составе Build 42.12 (версия для Linux).

Требуются только две нативные библиотеки:

  • libRakNet64.so
  • libZNetNoSteam64.so

Скачайте эти файлы из дистрибутива Linux Build 42.12, поместите оба файла в один .zip архив и загрузите этот архив при создании нового экземпляра игры.

Архив .zip должен содержать два файла .so в корне (не внутри папки).

Архив с нативными библиотеками можно загрузить вместе с основными файлами игры в соответствующих полях при создании нового экземпляра игры.

Рендереры и драйверы

Zomdroid использует рендереры для перевода десктопной графики Project Zomboid в графику, совместимую с Android-устройствами.

Лаунчер предоставляет два варианта рендеринга:

  • GL4ES
  • ZINK (ZINK_ZFA / ZINK_OSMESA)

GL4ES

GL4ES — наиболее стабильный рендерер для Build 41. Однако в Build 42 он нередко вызывает графические артефакты, поскольку не может полностью перевести обновлённый графический конвейер игры.

ZINK

ZINK обеспечивает наилучшие результаты с Build 42 и обычно устраняет графические артефакты, характерные для GL4ES.

ZINK предлагает несколько Vulkan-драйверов:

  • Системный Vulkan-драйвер
  • Freedreno
  • Turnip

Все драйверы Freedreno и Turnip — это сборки одного и того же открытого Mesa Vulkan-драйвера для Adreno GPU. Turnip — официальное название Vulkan-компонента в стеке Mesa Freedreno — эти названия иногда используются взаимозаменяемо. Разные сборки оптимизированы под разные поколения Snapdragon, поэтому лаунчер предлагает несколько вариантов драйверов.

ZINK в первую очередь работает на устройствах Snapdragon (GPU Adreno). На MediaTek Dimensity ZINK может работать через графический драйвер ANGLE, однако включить ANGLE на устройствах Dimensity зачастую непросто — через настройки разработчика это редко работает само по себе и может потребовать сторонних инструментов. На MediaTek Helio ZINK не поддерживается, однако некоторые устройства Helio могут запускать более ранние, менее требовательные версии Build 42 (например, 42.6) через GL4ES — надёжного решения пока нет, но ситуация может измениться в будущем. Для других процессоров (Exynos, Kirin и др.) информации мало — ZINK часто не работает или может вызывать сбои.

Рекомендуемая конфигурация для Build 41 — GL4ES. Для Build 42 рекомендуемая конфигурация — ZINK + драйвер Freedreno. Драйверы Freedreno поддерживаются только на устройствах Snapdragon с GPU Adreno. Разные версии драйверов предназначены для разных поколений железа:

  • Freedreno — стабильный, рекомендуется для Adreno 6xx и 7xx (Snapdragon до 7 Gen 1)
  • Freedreno 8xx — экспериментальный, для Adreno 8xx (Snapdragon 8 Gen 3 / Gen 4 / Gen 5 / Elite)
  • Freedreno 830/840 — экспериментальный, специально для Adreno 830/840 (Snapdragon 8 Elite / Elite 2, например Galaxy S25/S26)

Известные проблемы

Build 42.12+ — вылеты при генерации мира

Начиная приблизительно с Build 42.12, игра может падать во время создания мира — конкретно при загрузке в игру после создания персонажа. По всей видимости, это проблема на стороне игры, не связанная с устройством или драйвером — она наблюдается на разных устройствах и конфигурациях.

Это не постоянная ошибка. Если игра падает на этом этапе — просто перезапустите Zomdroid и попробуйте снова. Большинству пользователей требуется 2–4 попытки для первой успешной загрузки мира. После того как мир создан, последующие загрузки того же сохранения обычно проходят стабильно.

Совместимость графических модов

Некоторые графические моды — например, ретекстуры одежды, волос или транспортных средств — могут отображаться некорректно или не работать совсем. Это известное ограничение слоя перевода графики, используемого для запуска игры на Android, и оно не зависит от конкретного устройства или драйвера.

Исчезновение GUI рядом с разобранными машинами

Интерфейс игры может исчезнуть, если рядом находится транспортное средство, у которого снято 3 или более колёс. Причина — ошибка рендеринга, связанная с состоянием машины. Решение: не снимайте более 2 колёс с одного транспортного средства. Если интерфейс уже исчез — отойдите подальше от машины или перезагрузите сохранение.
